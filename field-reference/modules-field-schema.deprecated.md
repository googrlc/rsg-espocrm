# DEPRECATED: `modules-field-schema.deprecated.csv`

This field-schema report (and its generator `tools/build-field-schema-csv.py`) is
**deprecated**. Do not use it as a source of truth.

## Why
It is generated from `metadata/full-metadata.json`, a **stale camelCase snapshot**
that no longer matches the live server:

| | This (old) report | Live server |
|---|---|---|
| Field naming | camelCase (`accountStatus`) | snake_case (`account_status`) |
| Account fields | 365 | 279 |
| `db_type` column | metadata-derived | n/a |

## Use instead
**`exports/crm_fields_account_contact_policy.csv`** — regenerated by
`tools/build-crm-field-inventory.py` from a live read-only pull (merged
entityDefs + i18n + `information_schema` real DB column types). Covers Account,
Commission, Contact, Policy, Quote, Task.

## Caveat
The new report does **not yet** cover **Lead, Opportunity, Renewal** — those three
modules only exist in this deprecated file (also stale). Add them to the live
report before relying on them.
